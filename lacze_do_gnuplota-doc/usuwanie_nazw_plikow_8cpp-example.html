usuwanie_nazw_plikow.cpp

w kodzie poniżej zostało zademonstrowane, jak można wyszukiwać obiekt zawierający dane o pliku na podstawie nazwy pliku. Do tego celu wykorzystana została metoda PzG::LaczeDoGNUPlota::ZnajdzNazwePliku(). Występuje ona w czterech wariantach. W dalszej części przedstawione zostało, jak można usuwać wybrane nazwy plików. Do tej metody wykorzystano metodę PzG::LaczeDoGNUPlota::UsunNazwePliku(). Metoda ta występuje w trzech wariantach.

Ze względu na to, że kod programu zawiera dużo linii, najpierw został on omówiony. W dalszej części przedstawiona jest pełna zawartość programu.

W pierwszej części programu inicjalizowane są odpowiednie struktury, następnie wywołana jest metoda Rysuj().

int main()
{
PzG::LaczeDoGNUPlota Lacze;
Lacze.ZmienTrybRys(PzG::TR_3D);
Lacze.Inicjalizuj(); // Tutaj startuje gnuplot.
Lacze.UstawZakresX(-40, 100);
Lacze.UstawZakresY(-90, 90);
Lacze.UstawZakresZ(-20, 90);
Lacze.UstawRotacjeXZ(40,60); // Tutaj ustawiany jest widok
PzG::InfoPlikuDoRysowania *wInfoPliku1 = &Lacze.DodajNazwePliku("dat/prostopadloscian1.pow");
PzG::InfoPlikuDoRysowania *wInfoPliku2 = &Lacze.DodajNazwePliku("dat/prostopadloscian2.pow");
PzG::InfoPlikuDoRysowania *wInfoPliku3 = &Lacze.DodajNazwePliku("dat/prostopadloscian3.pow");
wInfoPliku2->ZmienKolor(3);
wInfoPliku2->ZmienSzerokosc(2);
wInfoPliku3->ZmienKolor(5);
wInfoPliku3->ZmienSzerokosc(4);
Lacze.Rysuj(); // Teraz powinno pojawic sie okienko gnuplota
// z rysunkiem, o ile istnieje plik "prostopadloscian3.pow"

Wywołanie metody Rysuj() powoduje pojawienie się okienka przedstawionego poniżej.

Zrzut ekranu pokazujący wynik działania programu.

Wykonanie kodu przedstawionego poniżej powoduje odnalezienie obiektu zawierającego informacje dotyczące trybu rysowania danych zawartych w pliku o nazwie "dat/prostopadloscian1.pow".

//----------------------------------------------------------------------------------------------
// Szukanie obiektu po nazwie pliku. Obiekt ten zawiera informacje dotyczące rysowania danych
// z pliku.
//
PzG::InfoPlikuDoRysowania *wInfoPlikuX = Lacze.ZnajdzNazwePliku("dat/prostopadloscian1.pow");
if (wInfoPlikuX == nullptr) {
cout << endl
<< ":( Dane o pliku nie zostaly znalezione. Cos poszlo nie tak." << endl
<< ":( Dzialanie programu zostalo przerwane." << endl
<< endl;
return 1;
}
cout << endl
<< ":) Dane o pliku zostaly znalezione." << endl
<< ":) Nazwa pliku: " << wInfoPlikuX->WezNazwePliku() << endl;
if (wInfoPliku1 != wInfoPlikuX) {
cout << endl
<< ":( Wskaznik zwrocony przez metode DodajNazwePliku() nie jest taki sam jak ten" << endl
<< ":( zwrocony przez metode ZnajdzNazwePliku(). Cos poszlo nie tak."
<< ":( Dzialanie programu zostalo przerwane." << endl
<< endl;
return 1;
}
cout << endl
<< ":) Wskaznik zwrocony przez metode DodajNazwePliku() jest taki sam jak ten" << endl
<< ":) zwrocony przez metode ZnajdzNazwePliku(). Wszystko jest tak jak powinno byc!" << endl
<< endl;
cout << "Nacisnij ENTER, aby zakonczyc ... " << flush;
cin.ignore(10000,'\n');

Naciśnięcie przycisku ENTER powoduje wykonanie następnej partii kodu, która wymusza usunięcie obiektu zawierającego nazwę pliku "dat/prostopadloscian2.pow".

//----------------------------------------------------------------------------------------------
// Teraz usuwamy nazwe z listy plikow, ktore zawieraja dane do rysowania
//
if (!Lacze.UsunNazwePliku("dat/prostopadloscian2.pow")) {
cout << ":( Operacja usuniecia nazwy \"dat/prostopadloscian2.pow\" nie powiodla sie." << endl;
cout << ":( Cos poszlo nie tak. Dzialanie programu zostalo przerwane" << endl;
return 1;
}
cout << ":) Operacja usuniecia nazwy \"dat/prostopadloscian2.pow\" powiodla sie." << endl
<< ":) Dzieki temu, w kolejnym wywolaniu metody Rysuj(), ten prostopadloscian nie narysowal sie." << endl
<< endl;
Lacze.Rysuj(); // Teraz ponownie odrysowuje się rysunek bez "dat/prostopadloscian2.pow"
cout << "Nacisnij ENTER, aby zakonczyc ... " << flush;
cin.ignore(10000,'\n');

Ponowne wywołanie metody Rysuj() powoduje odrysowanie rysunku. Tym razem jednak bez prostopadłościanu, którego opis znajdował się w pliku "dat/prostopadloscian2.pow".

Kolejne naciśnięcie przycisku ENTER powoduje wykonanie następnej partii kodu, w którym jest usuwany obiekt zawierający nazwę pliku "dat/prostopadloscian1.pow". Tym razem jest to zrealizowane z wykorzystaniem wskaźnika na ten obiekt.
Uwaga: Po wykonaniu tej operacji, zmienna wInfoPliku1 zawiera wskaźnik na nieistniejący obiekt. Oznacza to, że próby odwołania się do tego adresu będą skutkowały naruszeniem pamięci.

Omawiana partia kodu ma postać:

//----------------------------------------------------------------------------------------------
// Teraz usuwamy nazwe z listy plikow, ktore sa zawarte obiekcie klasy PzG::InfoPlikuDoRysowania
// Ten obiekt znajduje sie na liscie z ktorej zostanie usuniety.
//
if (!Lacze.UsunNazwePliku(wInfoPliku1)) {
cout << ":( Operacja usuniecia nazwy \"dat/prostopadloscian1.pow\" nie powiodla sie." << endl;
cout << ":( Cos poszlo nie tak. Dzialanie programu zostalo przerwane" << endl;
return 1;
}
cout << ":) Operacja usuniecia nazwy \"dat/prostopadloscian1.pow\" powiodla sie." << endl
<< ":) Dzieki temu, w kolejnym wywolaniu metody Rysuj(), ten prostopadloscian nie narysowal sie." << endl
<< endl;
Lacze.Rysuj(); // Teraz ponownie odrysowuje się rysunek bez "dat/prostopadloscia1.pow"

Wywołanie metody rysuj spowoduje, analogicznie jak w poprzednim przypadku, ukazanie się rysunku bez prostopadłościanu, którego opis znajdował się w pliku "dat/prostopadloscia1.pow".
